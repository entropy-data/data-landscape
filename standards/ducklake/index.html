Catalog APIs

DuckLake

Assess Vendor-led DuckDB Labs (vendor-driven open source) Since 2025 Single-vendor spec

Judgement: Assess

DuckDB-Labs catalog; v1.0 production-ready, ecosystem still small.

Lakehouse catalog format from DuckDB Labs that puts the catalog inside a regular SQL database — no separate REST catalog, no manifest sprawl. Designed for simplicity and zero-infrastructure analytics.

Why it counts as a standard

DuckLake's catalog schema is published as an open SQL specification — any database that speaks SQL can host a DuckLake catalog. Listed on the de-facto criterion: the schema is the standard surface, even though DuckDB Labs governs it and adoption is still early.

Example

The catalogue is an ordinary SQL database and the data is Parquet in object storage; DuckDB attaches both as one lakehouse.

INSTALL ducklake;

ATTACH 'ducklake:metadata.ducklake'
    AS my_ducklake
    (DATA_PATH 'data/');

USE my_ducklake;

CREATE TABLE orders (id BIGINT, city VARCHAR, amount DECIMAL(10,2));
